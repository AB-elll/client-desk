# ClientDesk 要件定義書

**バージョン**: 0.2.0
**作成日**: 2026-03-26
**ステータス**: ドラフト

---

## 1. システム概要

### 1.1 プロダクト名
**ClientDesk**

### 1.2 コンセプト
クライアント企業の経営情報（契約・許認可・従業員・資産等）を常時構造化管理し、M&Aデューデリジェンスにいつでも対応できる状態を維持するシステム。

### 1.3 DriveDeskとの関係
```
DriveDesk → 書類を検知・分類・抽出
    ↓ 構造化データを送信
ClientDesk → 台帳に蓄積・管理
    ↓
スプレッドシート / アラート / DDエクスポート
```
DriveDeskが「書類処理パイプライン」、ClientDeskが「データルーム」として機能する。

### 1.4 OSS方針
特定クライアントに依存しない汎用設計。設定ファイルで任意クライアントに対応。

**最初のユースケース**: 合同会社Kara's（薬局経営）

## 4. 機能要件

### 4.1 台帳管理（Registry）
- [ ] 全20カテゴリをSQLiteで構造化管理
- [ ] カテゴリごとに最適化されたスキーマ
- [ ] DriveDesKからの自動取込インターフェース
- [ ] 手動入力・更新インターフェース（初期: CLIまたはスプレッドシート直接編集）
- [ ] クライアントIDで複数クライアントを並行管理

### 4.2 期限アラート（Alert）
- [ ] 期限を持つ全レコードを一元監視
- [ ] アラートのタイミングは設定可能（例: 90日前・30日前・7日前）
- [ ] 通知先はプラグイン方式（Telegram / スプレッドシート等）
- [ ] 対応済みフラグで消化済みアラートを管理

### 4.3 可視化（Dashboard）
- [ ] Googleスプレッドシートに全カテゴリを同期
- [ ] カテゴリ別シート + サマリーシート
- [ ] 期限が近い項目を色付きで強調

### 4.4 共有・公開（Share）
**実装: Papermark**

- [ ] カテゴリ・範囲を指定してURL発行
- [ ] 共有相手ごとに閲覧権限を設定可能
- [ ] アクセスログ・閲覧状況を記録
- [ ] 期限付きURL（有効期限設定）

**想定される共有相手と権限**

| 共有相手 | 閲覧範囲 |
|---------|---------|
| クライアント代表者 | 自社情報の全カテゴリ |
| 税理士 | 財務・税務カテゴリのみ |
| 社会保険労務士 | 労務カテゴリのみ |
| 弁護士 | 法務・契約カテゴリのみ |
| M&A買収希望者 | 指定カテゴリのみ・期限付き |

> 毎回PDFを作って送る代わりに、常に最新の台帳をURLで共有する運用を実現する。

### 4.5 DDエクスポート（Export）
- [ ] カテゴリ指定または全件のエクスポート
- [ ] 出力形式: スプレッドシート（DDデータルーム用）
- [ ] エクスポート時点でのスナップショット生成

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| 汎用性 | 設定ファイルで任意クライアントに対応 |
| 拡張性 | カテゴリ追加・カスタムフィールド対応 |
| DriveDesk連携 | 標準インターフェースで自動取込 |
| セキュリティ | クレデンシャルは環境変数/.env管理 |
| 可観測性 | 全更新履歴をログに記録 |
| OSS | MITライセンス |

## 6. ユースケース詳細: 合同会社Kara's

### 6.1 前提
- 薬局経営（くるみ薬局）
- 管理者: 経営コンサル（翔太さん）
- 特記: 許認可（薬局開設許可・保険薬局指定・麻薬取扱）が事業継続の生命線

### 6.2 優先管理項目（Kara's固有）
1. 薬局開設許可証の有効期限
2. 薬剤師免許（従業員ごと）の更新期限
3. 麻薬取扱者免許・管理台帳
4. 保険薬局指定の更新
5. 各種リース・サービス契約の更新日
